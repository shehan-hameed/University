Executive Summary

The Smart Classroom Attendance & Environment System is a university-scale IoT and AI project designed to automate the most friction-heavy aspects of classroom management. By combining embedded hardware, cloud AI, and a suite of user-facing applications, the system removes the need for manual attendance taking, optimises the classroom environment in real time, and enhances campus security.

Deployed in a real university classroom at 1:1 scale, this system addresses three core institutional pain points: inefficient manual attendance, unnecessary energy consumption, and gaps in campus safety during emergencies.

The project is designed to keep all five team members fully occupied across two semesters, spanning hardware engineering, AI/ML development, cloud infrastructure, and multi-platform UI design.

Problem Statement

University classrooms continue to rely on outdated, manual processes for managing attendance, controlling environmental conditions, and ensuring student safety. These gaps result in measurable institutional costs:

  • Manual attendance is slow, error-prone and easily falsified
    Lecturers spend 5–10 minutes per session on roll calls; proxy attendance is common
  • Classrooms waste energy when unoccupied or under-occupied
    AC, lighting and fans run continuously regardless of actual occupancy
  • Emergency evacuation headcounts are unreliable
    No real-time record of who is in which room makes emergency response difficult
  • Chronic absenteeism goes undetected until it is too late
    No automated alert pipeline to students, parents or academic advisors
03

Project Objectives

The system is designed to deliver on the following primary objectives:

  • Automate attendance capture using AI-powered face recognition — eliminating manual roll calls entirely
  • Enforce time-based smart access control via a solenoid door lock integrated with the attendance system
  • Monitor and automatically control classroom climate (temperature, CO₂, light, noise) based on occupancy and sensor data
  • Deliver automated absence notifications to students and escalate to parents/guardians when thresholds are exceeded
  • Provide a live evacuation headcount dashboard for safety wardens during fire or emergency events
04

Feature Modules

The system is composed of six distinct feature modules, each forming a standalone workstream with cross-module integration at the cloud layer.

📷
Face Recognition Attendance

OV2640 camera at the classroom door captures faces. TinyML performs on-device detection; cloud AI (AWS Rekognition / DeepFace) handles recognition and matches against the registered student database. Attendance is logged, cross-referenced with the schedule, and late arrivals are flagged automatically.

🔐
Smart Access Control

A solenoid door lock is released only on verified face match. The door auto-locks after class begins. Any unrecognised entry attempt triggers a real-time security alert to the admin panel and logs the event with a timestamped image.

🌿
Environment Monitoring & Control

DHT22, MQ135, BH1750 and sound sensors feed live data to the ESP32-S3. A relay module automatically adjusts AC, fans and lighting based on ML-driven occupancy and comfort thresholds. All readings are logged to the cloud dashboard.

🔔
Late & Absence Notifications

Students receive an automated app notification or SMS if they are marked absent. A configurable threshold triggers escalation to parents or guardians. Persistent absence patterns are flagged to the academic warning system.

🏃
Evacuation & Emergency Headcount

When a fire or smoke alarm triggers, the system takes a snapshot of the current room occupancy. A live headcount dashboard is presented to safety wardens, with missing persons automatically flagged and a downloadable emergency report generated.

🧠
AI & ML Pipeline

On-device TinyML (ESP-WHO) handles real-time face detection. Cloud models handle recognition, environment optimisation, attendance-based dropout risk prediction, and anomaly detection for unauthorised access patterns.

Technology Stack

The system is built on a layered architecture — embedded firmware at the edge, MQTT for device communication, REST APIs for application integration, and cloud AI for heavy inference tasks.

Edge / Firmware
  • ESP32-S3 (Arduino / ESP-IDF)
  • ESP-WHO (TinyML face detection)
  • MQTT client (HiveMQ / EMQX)
  • OV2640 camera driver
  • DHT22, BH1750, MQ135 libraries

  • Firebase / AWS (database & hosting)
  • HiveMQ / EMQX (MQTT broker)
